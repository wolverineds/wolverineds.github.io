云计算初探

Veröffentlicht am 2017-05-15 | in blog

1.什么是云计算
那么什么是云计算呢? IBM 技术白皮书中云计算的定
义是 : “云计算一词用来描述一个系统平台或者一种类型的应用程序。一个云计算的平台按需提供动态的部署( provision) 、 配置( configuration) 、重新配置( reconfigure) 以及取消( deprovision) 等服务。”IDC［3］的云计算定义为: “ ( 云计算是) 一个新的 IT 开发和使用模型，云计算可以通过互联网实时地提供需要 的服务( 产品) 和解决方案。”对于云计算的定义，虽众说纷纭，但普遍认为云计算是通过互联网提供按需的、面向海量数 据处理和完成复杂计算的平台。事实上，云计算是网格计算 ( grid computing) 和公用计算( utility computing) 的进一步发展， 是对现有技术的高度抽象。云计算的孕育和发展离不开并行 计算技术、虚拟化、 SOA、容错技术、安全技术和海量数据处理 技术的发展，当然也离不开宽带网的普及。 云计算的资源和计算能力由专门的第三方提供商提供 ( 如 IBM、 Amazon 和谷歌等) ，用户无须顾虑软硬件的升级和维 护，对用户而言，云计算系统中复杂的“云”已经通过多层虚拟
化技术被完全屏蔽掉了。云计算的出现避免了用户在本地建 设、运行和维护价格昂贵的计算系统的费用，通过支付低廉的 服务费用，就可以完成在本地需耗费巨大的计算或处理任务。 低成本是云计算的一个显著特点。云计算其他的主要特点 如下: a) 云计算可以轻松实现不同设备间的数据与应用共享。 b) 云计算描述了一种可以通过互联网进行访问的可扩展 和动态重构的模式。 c) 云计算的应用使用大规模的数据中心以及功能强劲的 服务器来运行网络应用程序与网络服务，提供近乎实时的部署 和使用。 d) 任何一个用户通过合适的互联网接入设备以及一个标 准的浏览器就可访问一个云计算应用程序。云计算对用户端 的设备要求低，用户很少需要或不需要专门的 IT 技术。 e) 提供月租、订阅或当次付费的收费模式。

2 云计算的构成
1) 从分层角度看云计算的构成可以简单地把云计算系统看成由网络浏览器和“云”两部分组成。其中，网络浏览器可以是人们熟知的 IE、 Mozilla Firefox、 Google 浏览器等，这些浏览器通过网络连接到云，因而在 讨论云计算的构成时，关注的重点往往是“云”的构成。“云” 指的是系统中的计算机群，其规模和能力理论上不受限制，而 且可以在网络中的任意地方，如谷歌的网络服务( 包括搜索引 擎、谷歌地球等) 架构在由多达 200 多万台计算机构成的云计算平台之上， Amazon、微软、 IBM 等的云计算平台也都达到了几 十万台计算机的规模。数以万计的各种各样的计算机、服务器 和数据存储系统共同组成了云计算系统中的“云”，如 IBM 的 云体系结构［4］就包含了数据中心， IBM Tivoli Provisioning Manager、 IBM Tivoli Monitoring、 IBM WebSphere Application Server、 IBM DB2 和虚拟机。简化的云计算结构如图1 所示［5］，其中 中间件( middleware) 用来管理整个系统和满足用户的不同需求。
这样的云平台可以及时提供各种各样的服务和应用程序。 实际应用中，视不同情况，图 1 的详略程度也有所不同。 比如北大网络实验室开发的中国 Web 信息博物馆［8］( 也称为 Web InfoMall) ，就实现了如图 2 所示的三层云计算平台( 称为 Tplatform ［9］) 。其中，最底层提供硬件和必要的存储系统; 中间 层是该小组的研究重点，目前流行的谷歌的 Google file system ( GFS)［10］、 MapReduce ［11］和 Bigtable［12］在该层中都有所体现。 值得一提的是，北大网络实验室实现了与 GFS 类似的但主要 针对变长数据块的海量数据处理机制; 最上层是一些为用户提供的基本服务，用户可基于这些服务开发或集成不同的应用程序。
